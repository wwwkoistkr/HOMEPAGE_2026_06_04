# 🏗️ 슬라이더 통합 관리자 제어 시스템 — 실현 가능성 & 영향 분석 보고서

**작성일**: 2026-04-21
**버전**: v39.2.1 기준 설계안
**요청 사항**:
1. 슬라이더에 표시되는 **모든 숫자·문자**를 관리자 모드에서 새로운 저장소로 관리
2. 슬라이더 바의 **모든 색상**을 관리자 모드에서 수정 가능하게
3. 웹앱 수정 범위와 실현 가능성 분석
**원칙**: ❌ **코딩은 하지 않음** — 설계만 제시

---

## 1. 🎯 결론 (Executive Summary)

| 질문 | 답 |
|---|---|
| **가능한가?** | ✅ **100% 가능** |
| **얼마나 어려운가?** | ⭐⭐ **중간** (기존 시스템 대비 일관된 패턴 확장) |
| **웹앱 수정이 많은가?** | **보통** — 새 파일 2개 추가, 기존 파일 3개 수정 (약 **200~300줄**) |
| **소요 시간** | **2~3시간** (DB 마이그레이션 + API + UI + 배포) |
| **위험도** | ⭐ **낮음** — 기존 `site_settings` 테이블을 활용하거나 새 테이블 추가만 하면 됨 |
| **기존 데이터 영향** | **없음** — 기본값이 DB에 없으면 현재 하드코딩 값이 그대로 쓰임 (Fallback 패턴) |

### 한 줄 요약
> **이미 있는 `site_settings` 패턴을 그대로 따라 슬라이더 전용 설정만 추가**하면 되며, 기존 웹앱을 대규모로 수정할 필요 없이 약 200~300줄의 추가·수정으로 구현 가능.

## 3. 🏛️ 새 설계 — 슬라이더 통합 관리자 제어 시스템

### 3-1. 설계 철학 — 2가지 전략 비교

| 전략 | 방식 | 장점 | 단점 | 추천 |
|---|---|---|---|:---:|
| **A. site_settings 확장** | 기존 테이블에 `slider.*` 키 30~40개 추가 | • **코드 변경 최소화** (기존 API 활용)<br>• 마이그레이션 1개 파일만 추가<br>• 관리자 UI도 기존 설정 페이지에 바로 표시 | • 설정이 분산돼 관리 동선 길어짐 | ✅ **권장** |
| **B. 새 전용 테이블 생성** | `slider_config` 테이블 신규 | • 모든 슬라이더 설정이 한 곳에<br>• 전용 UI 페이지 구성 가능<br>• 프리셋(테마) 저장 기능 확장 용이 | • API 라우트·UI 페이지 신규 개발 필요<br>• 마이그레이션 추가 + admin.ts 추가 코드 | ⭐ **선택지** |

> **실무 추천**: **A+B 하이브리드** — 기본은 `site_settings`에 키를 두되, 관리자 UI에서 "🎨 슬라이더 설정"이라는 카테고리로 묶어 전용 페이지에 노출. DB는 단순, UI는 전용.

## 7. ⚠️ 리스크 및 고려사항

### 7-1. 주요 리스크

| 리스크 | 발생 가능성 | 영향도 | 완화책 |
|---|:---:|:---:|---|
| 색상 미입력 시 화면 깨짐 | 중 | 상 | **Fallback 기본값** 항상 유지 |
| 잘못된 HEX 입력 | 중 | 중 | Color Picker 강제 + 정규식 검증 |
| 텍스트 `{n}` placeholder 누락 | 저 | 하 | UI에서 검증 + 누락 시 현재값 표시 |
| CSP(Content Security Policy) 위반 | 저 | 상 | style 인라인 유지 (현 구조와 동일) |
| 캐시로 설정 반영 지연 | 중 | 저 | `?v=timestamp` 또는 `Cache-Control` |
| 관리자 실수로 모든 색이 검정 | 저 | 중 | **"기본값 복원" 버튼** 필수 |
| 기존 사용자 컨텐츠 파괴 | **없음** | — | 마이그레이션은 INSERT OR IGNORE |

### 7-2. 보안 고려

- 색상 값: HEX/RGB 정규식으로 검증 → XSS 방지
- 텍스트 포맷: `{n}` 외 HTML 태그 제거 → innerHTML 사용하는 일부 필드는 sanitize 필요
- 관리자 인증: 기존 `admin_users` 테이블 + JWT 그대로 사용 (변경 불필요)

### 7-3. 성능 영향

- DB 쿼리: `site_settings` 1회 조회 (기존과 동일) → **영향 0**
- SSR 렌더링: +34개 키를 object로 주입 → **<1ms 증가**
- 프론트 번들 크기: `admin-slider.tsx` ~10KB 증가 → **관리자 페이지에만 영향**
