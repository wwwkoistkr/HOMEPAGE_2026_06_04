## 1. 그룹웨어 → Gmail 발송 실패(반송) 진단

- **증상**: 그룹웨어(koist.kr)에서 Gmail로 발송 시 반송. Gmail 오류 `421-4.7.26 unauthenticated`.
- **원인**:
  - DKIM 서명 완전 부재 (셀렉터 11종 조회 → 전부 없음)
  - SPF 정렬(alignment) 실패 (Return-Path가 `@bouncehost`, koist.kr 아님)
  - DMARC 부재 (`_dmarc.koist.kr` 없음)
- **SPF 레코드 자체는 정상**:
  ```
  v=spf1 ip4:1.234.64.153 ip4:1.234.64.154 ip4:1.234.64.155 ip4:1.234.64.156 ip4:3.36.19.117 ip4:13.124.70.130 ~all
  ```
- **해결 방향**: 이지웹에 DKIM 활성화 + Return-Path 정렬 + DMARC 요청 → 받은 TXT를 비아웹 DNS에 등록.
- **연락처**: 이지웹 02-6949-2027 / ebizhelp@naonsoft.com

## 2. 웹 포워딩 (www.koist.kr → www.koist.ai.kr)

- **목표**: www.koist.kr 접속 시 신 홈페이지 www.koist.ai.kr 로 자동 연결. **메일(MX/SPF) 무영향 보장.**
- **방식**: 비아웹 "포워딩 서비스" → URL 포워딩(유동 포워딩, 연결주소로 변경) 설정.
  - 연결주소: `https://www.koist.ai.kr`
  - 주소표시: 연결주소로 변경(유동 포워딩) 선택
- **설정 결과**: "정상적으로 신청이 완료되었습니다" (2026-06-10 적용)

## 3. 안전 원칙 (재발 방지)

- 🔴 **MX(`gwspam.ebizworks.co.kr`) / SPF(TXT) 레코드는 절대 수정·삭제 금지.** 과거 메일 중단 사고 원인.
- ✅ 웹 포워딩은 별도 기능이라 메일 레코드와 독립 → 웹만 조정하면 메일 안전.
