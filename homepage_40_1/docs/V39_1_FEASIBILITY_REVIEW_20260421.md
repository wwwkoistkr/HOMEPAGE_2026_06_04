## 1️⃣ 데이터 파이프라인 건전성 (End-to-End 검증)

### 파이프라인 흐름도

```
[관리자 UI]                         [Cloudflare D1]                 [서버 SSR]                 [브라우저]
/admin/sim-cert-types    ──PUT──►   sim_cert_types 테이블  ──READ──►  home.tsx               ealData 객체
(월 단위 입력 가이드)                (koist_min/max_weeks)            simTypeToEal()          → simulate()
                                    (trad_min/max_weeks)              buildEalEntry()         → updateChart()
                                                                      buildOverallEntry()     → DOM 업데이트
```

### 단계별 검증 결과

**[단계 1] 관리자 입력 → DB 저장**
- `PUT /api/admin/sim-cert-types/:id` 정상 작동
- `updated_at` 타임스탬프 실시간 갱신 확인 (2026-04-21 01:24~01:25)
- 1주 단위 정수 입력 수용

**[단계 2] DB → 서버 SSR 계산**
- 매 요청마다 `sim_cert_types` 테이블 조회 (캐시 없음)
- `simTypeToEal(traditional, koist)` 함수가 한 자리 소수로 변환
- W2M = 4.345 표준화

**[단계 3] SSR → HTML 렌더링**
- `ealData` 객체가 `<script>` 태그 내 인라인 삽입
- 현재 DB값 계산 결과와 실제 프로덕션 HTML 값 **100% 일치** 확인

**[단계 4] 브라우저 → 슬라이더 인터랙션**
- `simulate()` 함수가 0~100% 슬라이더 값에 대해 `lerp()`로 보간
- 내부 계산은 풀 정밀도, 표시만 한 자리 반올림 (`fmtM`)
- `updateChart()`가 DOM의 bar·text·badge를 모두 갱신

## 4️⃣ 발견된 경미한 주의 사항 (수정 불필요)

### ⚠️ 주의 (1) 반올림 경계에서 변화가 2~3주 밀릴 수 있음

- 예: `traditional_max_weeks` 32→33→34주 중 33주에서는 반올림상 변화 없음.
- **영향도**: 미미 (1주 더 조정하면 반영됨)
- **대응**: 관리자에게 "변화가 작으면 2~3주 단위로 조정" 안내 → **UI 가이드로 이미 해결됨**

### ⚠️ 주의 (2) Cloudflare Edge Cache

- 저장 직후 수초~수분간 이전 HTML이 캐시될 수 있음.
- **대응**: 
  - URL에 `?v=타임스탬프` 붙여 우회 (이미 가이드에 포함)
  - 또는 관리자 페이지의 "캐시 새로고침" 버튼 사용 (기존 기능)

### ⚠️ 주의 (3) `unified_reduction_default` 값이 35로 수동 설정됨

- 현재 관리자가 수동으로 `35`를 입력해 놓아 Hero Badge가 35%로 고정 표시됨.
- **서버 계산값**(현재 약 70%)이 더 정확하므로 이 필드를 **비우는 것을 권장**.
- **코드 수정 불필요** – 관리자가 `/admin` 콘텐츠 설정에서 값을 지우기만 하면 됨.

## 5️⃣ 운영 권장 가이드라인 (코드 수정 없음)

### ✅ 관리자가 할 일
1. `/admin` 로그인 → 사이드바 **"🤖 AI 시뮬레이터"** 클릭
2. **EAL2 / EAL3 / EAL4** 행만 수정 (KCMVP·보안기능확인서·성능평가는 시뮬레이터 탭에 없으므로 무시)
3. 각 행에서 4개 값을 입력:
   - **CCRA 최소(`traditional_min_weeks`)**: 사전준비가 완벽할 때 최단 주수
   - **CCRA 최대(`traditional_max_weeks`)**: 사전준비가 미흡할 때 최장 주수
   - **KOIST 최소(`koist_min_weeks`)**: KOIST 프로세스 최단 주수
   - **KOIST 최대(`koist_max_weeks`)**: KOIST 프로세스 최장 주수
4. **저장** 후 홈페이지에 `?v=새숫자` 붙여 확인 (예: `https://koist-website.pages.dev/?v=20260421a`)

### ✅ (선택) Hero Badge 자동 계산을 쓰려면
- `/admin` → 홈 콘텐츠 설정 → **"기본 단축률"** 입력란을 **비움**
- Badge가 실시간 서버 계산값(현재 ~70%)으로 자동 표시됨
